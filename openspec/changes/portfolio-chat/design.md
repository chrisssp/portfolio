# Design: Cooper — AI Portfolio Chatbot

## Technical Approach

Four-layer architecture: build-time content extraction → server-side API with keyword RAG → streaming Gemini 2.5 Flash → lazy-loaded client widget. The system is purely additive — no existing files are modified except `package.json` (new deps). Defense is server-side enforced since the repo is public.

## Architecture Decisions

| Decision | Options | Tradeoff | Choice |
|----------|---------|----------|--------|
| Retrieval strategy | Vector DB vs keyword+section match | Complexity/infra vs simplicity | Keyword+section match — zero infra, sufficient for portfolio scale |
| Response streaming | REST full-response vs SSE via AI SDK | Simplicity vs UX latency | `streamText` from `ai` package — real-time tokens, proven Vercel integration |
| Session state | localStorage vs sessionStorage | Persistence vs privacy | `sessionStorage` — clears on tab close, privacy-appropriate for chat |
| Widget loading | Eager vs `next/dynamic` ssr:false | Bundle size vs latency | Lazy dynamic import — chat is non-critical, reduces initial bundle |
| Content source | Runtime fetch vs build-time JSON | Freshness vs simplicity | Build-time `portfolio-content.json` — content changes only on deploy |
| Mobile strategy | Responsive CSS vs separate component | Code reuse vs UX fidelity | Single component with breakpoint-based layout — `>= 768px` floating, `< 768px` fullscreen |

## File Changes

| File | Action | Description |
|------|--------|-------------|
| `scripts/generate-content-json.ts` | Create | Node script: imports i18n modules, flattens bilingual content into `portfolio-content.json` with section tags |
| `public/portfolio-content.json` | Create | Build artifact — flat array of `{id, section, locale, title, description, techStack, ...}` |
| `app/api/chat/route.ts` | Create | POST handler: rate limiting (10/min/IP), pre-flight keyword filter, keyword+section RAG, `streamText` with Gemini 2.5 Flash, defense system prompt |
| `components/organisms/ChatWidget.tsx` | Create | Lazy-loaded chat component: bubble (bottom-center), panel (floating desktop, fullscreen mobile), message list, input, streaming render, session persistence |
| `components/organisms/ChatWidget/greetings.ts` | Create | Curated bilingual greeting array (6-8 entries), random per session |
| `package.json` | Modify | Add `@ai-sdk/google`, `ai` to dependencies; add `predev`/`postbuild` script for content generation |

## Interfaces / Contracts

```typescript
// Content chunk (generated by script, consumed by API)
interface ContentChunk {
  id: string;           // e.g. "7dcompass-en"
  section: "project" | "experience" | "education" | "about" | "hero" | "skills";
  locale: "en" | "es";
  title: string;
  description: string;
  fullDescription?: string;
  techStack?: string[];
  challenge?: { description: string; solution: string };
}

// API request
interface ChatRequest {
  messages: Array<{ role: "user" | "assistant"; content: string }>;
  locale: "en" | "es";
}

// Widget props (passed from layout via params)
interface ChatWidgetProps {
  locale: Locale;
}
```

## Positioning Strategy

Existing elements: MusicPlayer `bottom-6 left-6 z-[60]`, ScrollToTop `bottom-6 right-6 z-50`. Chat bubble: `fixed bottom-6 left-1/2 -translate-x-1/2 z-[60]` — centered between them. On mobile (`< 375px`), all three share the bottom row; chat bubble shifts up with `bottom-20` if needed. Chat panel: desktop `fixed bottom-20 left-1/2 -translate-x-1/2 w-[380px] h-[520px]`, mobile `fixed inset-0 z-[70]`.

## Animation Strategy

- **Panel open**: CSS `scale(0.95)→scale(1)` + opacity fade, 200ms ease-out. Mobile: `translateY(100%)→translateY(0)` slide-up.
- **Panel close**: reverse of open, 200ms.
- **New message**: `translateX(±12px)→0` + opacity, 250ms, user slides from left, Cooper from right.
- **Typing indicator**: three-dot bounce with staggered `animation-delay`.
- **Streaming cursor**: CSS `@keyframes blink` on a `|` character, toggled off on stream complete.
- **prefers-reduced-motion**: all transitions become `duration-0` via `motion-safe:` Tailwind variants.

## Sound Implementation

No external audio files. Two short tones generated programmatically via Web Audio API (OscillatorNode):

```typescript
function playSendTone() {
  const ctx = new AudioContext();
  const osc = ctx.createOscillator();
  osc.type = "sine";
  osc.frequency.setValueAtTime(800, ctx.currentTime);
  osc.frequency.exponentialRampToValueAtTime(600, ctx.currentTime + 0.08);
  osc.connect(ctx.destination);
  osc.start(); osc.stop(ctx.currentTime + 0.08);
}

function playReceiveTone() {
  const ctx = new AudioContext();
  const osc = ctx.createOscillator();
  osc.type = "sine";
  osc.frequency.setValueAtTime(600, ctx.currentTime);
  osc.frequency.exponentialRampToValueAtTime(1000, ctx.currentTime + 0.1);
  osc.connect(ctx.destination);
  osc.start(); osc.stop(ctx.currentTime + 0.1);
}
```

No files to download, no license concerns, zero network requests. Default muted, toggle stored in `sessionStorage('chat-sound-enabled')`. Send tone plays on message dispatch, receive tone plays on first token arrival.

## Defense Architecture

```
User message
    │
    ▼
┌─ Pre-flight keyword filter ─────────────────┐
│ Portfolio keywords? ──yes──► Continue        │
│                └──no──► Redirect response    │
│                        (no Gemini call)      │
└──────────────────────────────────────────────┘
    │
    ▼
┌─ Rate limiter (in-memory Map<IP, timestamps>) ─┐
│ >10/min? ──yes──► 429 response                 │
│            └──no──► Continue                    │
└─────────────────────────────────────────────────┘
    │
    ▼
┌─ Gemini safety settings ───────────────────────┐
│ BLOCK_MEDIUM_AND_ABOVE on:                     │
│ HATE_SPEECH, HARASSMENT, SEXUALLY_EXPLICIT,    │
│ DANGEROUS_CONTENT                              │
│ Blocked? ──yes──► "Let's keep it professional" │
└─────────────────────────────────────────────────┘
    │
    ▼
┌─ System prompt enforcement ────────────────────┐
│ Cooper persona + scope boundaries              │
│ 3-redirect max → session end                   │
│ Offensive content → immediate termination      │
└─────────────────────────────────────────────────┘
```

## Testing Strategy

| Layer | What to Test | Approach |
|-------|-------------|----------|
| Unit | Content extraction script output shape | Run script, validate JSON schema |
| Unit | Pre-flight keyword filter logic | Test keyword match/miss cases |
| Unit | Rate limiter window behavior | Mock timestamps, verify 429 |
| Integration | API route end-to-end | POST with mock messages, verify streaming response |
| Integration | RAG keyword matching | Test project/tech/section queries against generated JSON |
| E2E | Chat widget open/close/responsive | Manual test at desktop and mobile breakpoints |
| E2E | Session persistence | Send messages, navigate, verify restore |
| E2E | Defense: redirect, termination | Send out-of-scope, verify redirect; send offensive, verify termination |

## Migration / Rollout

No migration required. Pure additive feature. Content JSON is a build artifact (gitignored or committed — either works). Deploy as-is; API route becomes live once `GOOGLE_GENERATIVE_AI_API_KEY` env var is set in Vercel.
